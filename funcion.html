¿Como Funciona?

La clave principal de las aplicaciones web son los llamados servicios workers. Este es un script (el proxy controlable que se debe ejecutar entre el navegador y el servidor).


Los service workers forman parte de la base estándar PWA, utilizan el c caché web para obtener mejores resultados de manera instantánea. Los service workers estos duran mientras se quiere su acción.

En una PWA, cuando se hace clic en algún icono o se usa una función es decir uno de ellos entra en acción, se procesa decidiendo si la cache sin tener conexión puede completar la solicitud que se le ha pedido.

¿En que ayuda la Caché?

La idea es que tenga múltiples caches fuera de línea para que PWA la selecciones, el cual brinda una gama mucho más amplia de tener funcionalidades sin conexión. La caché no solo se utiliza para el aumento de velocidad fuera de línea.

Si nosotros ejecutamos una PWA, pero nuestra conexión es extremadamente irregular el service workers este puede servir una caché anterior, en pleno funcionamiento, sin interrumpir la experiencia.

En la actualidad hay sitios web que son tan bien implementados, que cargan de una manera muy rapidísima, incluso mucho más rápido que una app nativa.

Existen muchas tecnologías de uso común para crear aplicaciones web progresivas. Todas las PWA requieren, como mínimo, un service worker y un manifiesto.
